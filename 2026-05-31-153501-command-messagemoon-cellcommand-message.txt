  5. 암호화 스택 (E2EE Sync)

  BIP39 복구 구문 → argon2 키유도 → ChaCha20-Poly1305 암호화
                    → ed25519-dalek 서명 → blake3 해시
                    → SHA-256 보조

  - 워크스페이스별 독립 키
  - 서버는 암호화된 봉투만 보관 (평문 접근 불가)
  - 프리사인 URL로 S3에 직접 업로드/다운로드

  7. 아키텍처 강점

  1. Local-first 원칙: 파일은 로컬 Markdown, 클라우드는 선택사항
  2. E2EE by design: 서버가 평문 절대 접근 불가 (proto에 명시)
  3. 플러그인 아키텍처: 14개 빌트인 + 확장 가능한 시스템
  4. 프로토콜 계약: protobuf 기반 타입 안전 RPC — Go/TS/Rust 3언어 코드 생성
  5. AI 리뷰 가능: AI가 제안하면 diff로 보여주고 승인 후 적용
  6. Second Brain: 메모리 + 위키 + 의사결정문서가 Markdown으로 관리
  7. 모노레포 일관성: moonrepo로 전체 빌드 파이프라인 통합

  2. AI 철학
  - mnemosyne: AI가 추출/구조화 담당 (엔티티, 관계, 위키 페이지 생성). 사용자는
  결과를 검토.
  - Kuku: AI가 편집 담당 (Agent/Ask/Inline). 사용자가 diff로 승인/거부.
  근본적으로 다른 AI 활용 방식.

  3. 그래프 깊이
  - mnemosyne: 스키마 기반 엔티티-관계 그래프 (function→calls, statute→clause).
  구조화된 지식.
  - Kuku: wikilink 기반 문서 간 연결 그래프. 문서 간 탐색. 깊이가 다름.
