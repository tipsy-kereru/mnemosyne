● pyproject.toml에 이미 진입점이 설정되어 있어서 배포 준비는 거의 다 되어
  있습니다.

  현재 상태:

  [project.scripts]
  mnemosyne = "mnemosyne.cli:main"
  mnemosyne-query = "mnemosyne.graph.cli:main"
  mnemosyne-add = "mnemosyne.ingest.cli:add_main"
  mnemosyne-update = "mnemosyne.ingest.cli:update_main"

  pip으로 설치하면 자동으로 이 명령어들이 등록됩니다.

  배포 방법 (난이도 순):

  1. GitHub에서 바로 설치 (지금 가능)

  pip install git+https://github.com/tipsy-kereru/mnemosyne.git

  2. PyPI 배포 (표준 방법)

  # 빌드
  pip install build
  python -m build

  # 테스트 업로드
  pip install twine
  twine upload --repository testpypi dist/*

  # 정식 배포
  twine upload dist/*

  그러면 누구나:
  pip install mnemosyne-kg
  mnemosyne add --text "Hello" --domain daily

  3. 선택적 의존성 분리

  현재 LLM/URL 기능이 anthropic, openai 등 무거운 패키지를 optional로 두고
  있어서, 사용자가 필요한 것만 설치 가능합니다:

  pip install mnemosyne-kg                    # 기본 (LLM 없이)
  pip install mnemosyne-kg[ingest]            # LLM + URL fetch
  pip install mnemosyne-kg[deterministic]     # tree-sitter 코드 파싱
  pip install mnemosyne-kg[all]              # 전체

  PyPI 배포 전 필요한 것:
  - PyPI 계정 생성 (pypi.org)
  - twine upload 시 API 토큰 인증
  - 버전 태깅 (git tag v0.1.0)

  PyPI 배포 진행할까요?
